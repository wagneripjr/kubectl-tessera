# `kubectl-tessera` — Implementation Plan (Go, open-source)

> Handoff spec for Claude Code. Build the tool described here in **Go**, targeting an **open-source release on GitHub + krew**. Working name is **`tessera`** — it appears throughout; if you pick a different name, change the module path, binary name, krew name, and label domain in one find/replace pass. "Done" = the **Acceptance criteria** pass.

## What it is

A kubectl plugin invoked as `kubectl tessera` that issues an **ephemeral, scope-narrowed, TTL-bound** credential for the current cluster, running **as the invoking user**, with a `SelfSubjectRulesReview` / `SelfSubjectAccessReview` pre-flight and **automatic cleanup** of the RBAC objects it creates. It emits a throwaway kubeconfig you point Claude Code (or any client) at for a time-boxed, least-privilege session.

---

## Read this first: what this tool actually buys you

The minted token is **always** narrow — whoever uses it is confined to the scope you asked for. What changes is the guarantee around *your own* privilege:

- **If the operator is NOT cluster-admin:** Kubernetes RBAC escalation-prevention makes this a real security boundary. You physically cannot mint a token with more than you already hold — the API server rejects creation of a binding to a role whose rules you don't possess (unless you hold the `bind` verb on it).
- **If the operator IS cluster-admin (probably the case here):** escalation-prevention is vacuous, so the tool can't stop you from over-scoping. It is still an effective **self-imposed blast-radius guardrail for the agent**, but it is *not* a containment control on a privileged human. The hard backstop against destructive operations stays in **admission policy** (ValidatingAdmissionPolicy / Kyverno), not here.

This must be stated plainly at the top of the README. The tool is primarily an *accident-limiter for the agent*, and a real security boundary only for non-admin operators.

## Decisions locked (don't re-litigate)

- **Go + client-go**, Cobra for commands, `k8s.io/cli-runtime` for standard config flags. No shelling out to `kubectl`.
- **License: Apache-2.0.** It's the Kubernetes-ecosystem norm, krew-index is overwhelmingly Apache-2.0, and it carries an explicit patent grant — the right default for infra tooling you may upstream.
- **Distribution: krew + goreleaser.** goreleaser builds the cross-platform archives + checksums and generates the krew manifest; submit a PR to `kubernetes-sigs/krew-index` for discoverability.
- **Fail-safe defaults:** verbs default to `get,list,watch`; scope defaults to namespaced (never cluster-wide without `--cluster`); TTL defaults to `15m`.
- **Creation runs as the invoking user.** Never switch to a privileged context or use impersonation to create objects — the non-admin security property depends on this.
- **Two cleanup mechanisms, both shipped:** a foreground signal trap (for `--exec`) and a label/annotation + `tessera gc` sweep (for `--print-kubeconfig` and crash recovery).

## Architecture (Go)

- **Module path:** `github.com/<org>/kubectl-tessera`
- **Binary:** `kubectl-tessera` (krew/kubectl plugin convention → invoked as `kubectl tessera`).

## CLI surface

Cobra root command is `mint` behavior; subcommands `gc`, `ls`.

```
kubectl tessera \
  --verb get,list,watch          # default get,list,watch
  --resource pods,deployments    # required
  --namespace prod               # via ConfigFlags -n; comma list / repeatable for multi-ns
  --ttl 30m                      # Go duration; default 15m
  [--resource-name foo,bar]      # narrow to named objects (SSAR with name)
  [--api-group apps]             # auto-resolved via RESTMapper if omitted
  [--cluster]                    # cluster-scoped → ClusterRole + ClusterRoleBinding
  [--exec]                       # spawn subshell with KUBECONFIG; cleanup on exit (DEFAULT, interactive)
  [--print-kubeconfig]           # print kubeconfig path to stdout, leave objects for gc (USE for Claude Code)
  [--context NAME] [--kubeconfig PATH]   # from ConfigFlags
  [--dry-run]                    # pre-flight + print intended objects; create nothing
  [-o json]                      # machine-readable session output
```

`--exec` and `--print-kubeconfig` are mutually exclusive; default to `--exec`.

- `kubectl tessera gc` — delete expired managed objects across all namespaces. Idempotent, cron-safe.
- `kubectl tessera ls` — list active sessions (session-id, owner, scope summary, expires-at) from the managed objects' labels/annotations.

## Execution flow (`mint`) — client-go specifics

1. **Resolve config + clientset** from `ConfigFlags`. Capture the source context's cluster (server + CA) for the kubeconfig step later.
2. **Parse + validate scope.** Build a RESTMapper from discovery (`restmapper.GetAPIGroupResources` → `NewDiscoveryRESTMapper`). For each resource, resolve GVR/GVK and read `mapping.Scope.Name()` to determine namespaced vs cluster-scoped. Reject `--namespace` with a cluster-scoped resource (tell the user to use `--cluster`); require `--api-group` only when a resource is ambiguous across groups.
3. **Pre-flight gate (SSAR).** For every `(verb × resource × namespace[ × name])`, create an `authorizationv1.SelfSubjectAccessReview` with `Spec.ResourceAttributes` and check `.Status.Allowed`. Collect denials; if any, print an allowed/denied table and exit non-zero **before creating anything**. (No `--force`.)
4. **Generate identifiers.** `sessionID` = first 8 chars of a UUID, lowercased. Base name `tessera-<sanitized-owner>-<sessionID>` (DNS-1123). Compute `expiresAt = now + ttl` (RFC3339 UTC).
5. **Create RBAC objects as the user, with rollback.** Order: `ServiceAccount` → `Role`/`ClusterRole` → `RoleBinding`/`ClusterRoleBinding`, via the typed clientset (`rbacv1`). Stamp every object with the labels/annotations below. Track created objects; on any error, delete them in reverse order (foreground propagation) and return.
6. **Mint the token (TokenRequest API).**
   ```go
   tr := &authenticationv1.TokenRequest{
     Spec: authenticationv1.TokenRequestSpec{ExpirationSeconds: ptr.To(int64(ttl.Seconds()))},
   }
   out, err := clientset.CoreV1().ServiceAccounts(ns).CreateToken(ctx, saName, tr, metav1.CreateOptions{})
   // use out.Status.Token and out.Status.ExpirationTimestamp
   ```
   Use the **returned** `ExpirationTimestamp` — it reflects clamping by the cluster's `--service-account-max-token-expiration`. If it's shorter than requested, warn.
7. **Build the throwaway kubeconfig** (see section). Write `0600` to `${XDG_RUNTIME_DIR:-/tmp}/tessera/<sessionID>.kubeconfig`.
8. **Output mode:**
   - `--print-kubeconfig`: print only the kubeconfig path to **stdout** (all logs → stderr), exit 0. Objects persist; gc reclaims them; token self-expires.
   - `--exec`: `os/exec` a `${SHELL:-/bin/bash} -i` with `KUBECONFIG` set; `signal.NotifyContext` on `SIGINT/SIGTERM`; on subshell exit, delete the object set and remove the kubeconfig file.
9. **Audit line to stderr** on every mint: session-id, owner, scope, ttl, effective-expiry, namespace(s), cluster.

## Pre-flight: SSRR for discovery, SSAR for the gate

This is the part that's easy to get wrong — keep the split:

- **`SelfSubjectAccessReview`** is the **authoritative gate**. One yes/no per requested rule, including `name` scope. This is what blocks an over-ask before any object is created. (Equivalent to `kubectl auth can-i <verb> <resource>`.)
- **`SelfSubjectRulesReview`** is for **discovery only** (`--dry-run`/`--explain`: "here's what you could scope to in this namespace"). It is **not** a safe gate: surface its `.Status.Incomplete` flag — for non-enumerable authorizers (e.g. webhook) the rule list can be incomplete, and it does not evaluate `resourceNames`. (Equivalent to `kubectl auth can-i --list`.)

Both via `clientset.AuthorizationV1().SelfSubjectAccessReviews().Create(...)` and `...SelfSubjectRulesReviews().Create(...)`. The UX gate fails fast and clearly; the API server still enforces the real boundary at binding creation for non-admins.

## RBAC objects + labels

Namespaced (default): `ServiceAccount` + `Role` + `RoleBinding` in the target namespace.
Cluster (`--cluster`): `ServiceAccount` (in a configurable namespace, default `kube-system` — document it) + `ClusterRole` + `ClusterRoleBinding`.

Rules built from resolved `(apiGroups, resources, verbs[, resourceNames])`, one `PolicyRule` per apiGroup grouping. Core group = `""`.

**Labels (every object):**
```
app.kubernetes.io/managed-by: kubectl-tessera
tessera.adustio.io/owner: <sanitized-owner>
tessera.adustio.io/session-id: <sessionID>
```
**Annotation (every object):**
```
tessera.adustio.io/expires-at: <RFC3339-UTC>
```
> `tessera.adustio.io/` is a placeholder label domain — swap for your own DNS subdomain. Identical `session-id` across the set is what lets `gc`/`ls`/cleanup operate on a whole session atomically.

## Throwaway kubeconfig (clientcmd)

Requirements: same cluster `server` + CA as the source context, an AuthInfo holding **only** the token, a Context wired to the target namespace, file mode `0600`, isolated path. Never write the token to the user's main kubeconfig; never pass it as a process arg.

Build a `clientcmdapi.Config` (`k8s.io/client-go/tools/clientcmd/api`): copy the current cluster entry (preserve `CertificateAuthorityData` and any proxy URL), add `AuthInfos["tessera"] = {Token: token}`, add a `Contexts["tessera"]` binding cluster→authinfo with `Namespace`, set `CurrentContext`. Write with `clientcmd.WriteToFile`, then `os.Chmod(path, 0600)`.

## Cleanup

Three layers, defense-in-depth:

1. **Token TTL** — the API server auto-revokes at the returned `ExpirationTimestamp`. Primary protection; needs nothing from us.
2. **Foreground trap (`--exec`)** — on subshell exit or `SIGINT/SIGTERM`, delete the session's object set and `os.Remove` the kubeconfig file. Covers the interactive happy path and shell crashes.
3. **`tessera gc`** — list objects with `app.kubernetes.io/managed-by=kubectl-tessera` across namespaces, parse `expires-at`, delete sets where `now > expires-at`. Idempotent. **Required for `--print-kubeconfig`** (no foreground process to trap). Wire to host cron or ship the CronJob below.

`SIGKILL` bypasses the trap → objects orphaned → `gc` reclaims them next run. Document this.

### Optional in-cluster gc CronJob (`deploy/gc-cronjob.yaml`)
Every 5 min, minimal `kubectl`/distroless image running the `gc` logic, bound to a dedicated SA whose only permissions are `list`/`delete` on `serviceaccounts`, `roles`, `rolebindings`, `clusterroles`, `clusterrolebindings`, filtered by the managed-by label. Recommended for anyone using `--print-kubeconfig` regularly.

## Edge cases (handle all)

- **Operator lacks `create` on serviceaccounts/roles/rolebindings** → clear error naming the missing verbs; explain an admin must grant `create` on those (or `bind` on a curated role). Real gotcha for non-admins.
- **Requested TTL > cluster max** → token clamped; warn using the returned `ExpirationTimestamp`.
- **Partial creation failure** → reverse-order rollback; no orphans.
- **Namespaced vs cluster-scoped mismatch** → reject and instruct.
- **Ambiguous resource across apiGroups** → require `--api-group`.
- **Multiple namespaces** → one object set per namespace; document.
- **stdout hygiene** in `--print-kubeconfig` → only the path on stdout so `export KUBECONFIG=$(kubectl tessera … --print-kubeconfig)` works.
- **No token leakage** → never in argv, main kubeconfig, or shell history; kubeconfig `0600`.
- **k8s < 1.24** (no TokenRequest) → fail clearly. Target is 1.34.

## Hard "do NOT" list (these void the design)

1. **Do NOT** create objects with a privileged context or via impersonation — create as the invoking user.
2. **Do NOT** use `SelfSubjectRulesReview` as the gate. Gate with `SelfSubjectAccessReview`. SSRR is discovery only, and honor its `Incomplete` flag.
3. **Do NOT** leave orphaned objects on partial failure.
4. **Do NOT** default to wide scope.
5. **Do NOT** write the token to `~/.kube/config` or pass it on the command line.

## How this plugs into Claude Code

Mint a scoped, time-boxed kubeconfig, then launch Claude Code pointed at it:

```bash
export KUBECONFIG="$(kubectl tessera \
  --verb get,list,watch \
  --resource pods,deployments,events,replicasets \
  --namespace prod --ttl 1h --print-kubeconfig)"

claude   # read-only in prod, auto-expiring in 1h
```

Scoped-write incident (widen verbs deliberately, shorten TTL, pin the object):

```bash
export KUBECONFIG="$(kubectl tessera \
  --verb get,list,watch,patch,delete \
  --resource pods --namespace prod \
  --resource-name stuck-pod-abc123 --ttl 20m --print-kubeconfig)"
```

Ensure `tessera gc` (or the CronJob) runs, since `--print-kubeconfig` leaves objects for the sweeper.

## Acceptance criteria (the e2e matrix — "done" means these pass on kind)

Verify with `kubectl --kubeconfig <minted> auth can-i …` and real calls.

1. **Read-only scope:** `get,list,watch` on `pods` → can `get pods`, cannot `delete pods`.
2. **Pre-flight refuses over-ask (non-admin):** as a limited user, request a verb you lack → aborts before creating anything, prints allowed/denied table, non-zero exit.
3. **TTL expiry:** after TTL, minted token rejected (401/403).
4. **`--exec` cleanup:** exit subshell → SA/Role/RoleBinding gone, kubeconfig file removed.
5. **Crash recovery:** `kill -9` the exec process → orphans reclaimed by `tessera gc`.
6. **`--print-kubeconfig` hygiene:** `export KUBECONFIG=$(…)` yields a usable scoped kubeconfig, nothing but the path on stdout.
7. **Cluster scope:** `--cluster get,list nodes` → ClusterRole/Binding; can `get nodes`, cannot `delete nodes`.
8. **resourceName narrowing:** `--resource-name foo` → acts on `foo`, denied on `bar`.
9. **Rollback:** inject a mid-creation failure → no orphaned objects.
10. **gc selectivity:** deletes only expired, managed objects; never touches unmanaged RBAC or unexpired sessions.
11. **SSRR Incomplete surfaced:** `--dry-run` against a webhook-authorizer cluster prints a clear "discovery may be incomplete" notice.
