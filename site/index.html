Hand your agent

kubectl‑tessera mints an ephemeral, scope‑narrowed, TTL‑bound credential for the current cluster — running as you, gated by a pre‑flight authorization check, and cleaned up automatically.

tessera — a Roman token of scoped access, cut to grant exactly the entry it was made for, then spent. This mints the modern equivalent: a kubeconfig confined to the verbs and resources you ask for, that dissolves when its time is up.

What this tool actually buys you

The minted token is always narrow — whoever uses it is confined to the scope you asked for. What changes is the guarantee about your own privilege.

non‑admin operator

A real security boundary.

Kubernetes RBAC escalation‑prevention means you physically cannot mint a token with more than you already hold — the API server rejects binding to a role whose rules you don’t possess. The boundary is enforced by the cluster, not just by us.

cluster‑admin operator

An accident‑limiter, not containment.

If you already hold everything, escalation‑prevention is vacuous — the tool can’t stop you from over‑scoping. It’s still an effective blast‑radius guardrail for an agent, but the hard backstop against destructive operations is admission policy (ValidatingAdmissionPolicy / Kyverno).

Plainly: tessera is primarily an accident‑limiter for the agent, and a real security boundary only for non‑admin operators. It does not — and cannot — contain a malicious admin.

Install

Four ways to get the plugin

Installed as kubectl-tessera on your $PATH, it’s invoked as kubectl tessera.

A. krew — once accepted into the central index
kubectl krew
$ kubectl krew install tessera   # pending krew-index acceptance
B. go install — always available
go ≥ 1.24
$ go install github.com/wagneripjr/kubectl-tessera/cmd/kubectl-tessera@latest
C. install script — no krew, no Go; sha256‑verified (cosign when present)
linux / macOS — inspect, then run
$ curl -fsSLO https://raw.githubusercontent.com/wagneripjr/kubectl-tessera/master/install.sh
$ less install.sh   # it mints cluster creds — trust before you run
$ bash install.sh   # → /usr/local/bin (or ~/.local/bin)
D. manual — download, extract, drop on PATH
linux / macOS
$ tar -xzf kubectl-tessera_*_linux_amd64.tar.gz
$ sudo install kubectl-tessera /usr/local/bin/
$ kubectl tessera version
Usage

Scope it. Time‑box it. Walk away.

Defaults fail safe: get,list,watch, namespaced, 15‑minute TTL. You widen scope explicitly — never by accident. New here? Run kubectl tessera with no arguments for help and examples.

the simplest start — read‑only pods, current namespace
$ kubectl tessera --resource pods   # interactive shell, get/list/watch, 15m TTL
ephemeral cluster‑wide reader — every resource type
$ export KUBECONFIG="$(kubectl tessera \
    --resource '*' --all-namespaces \
    --ttl 1h --print-kubeconfig)"
point an agent at prod, read‑only, for an hour
$ export KUBECONFIG="$(kubectl tessera \
    --verb get,list,watch \
    --resource pods,deployments,events,replicasets \
    --namespace prod --ttl 1h --print-kubeconfig)"

$ claude   # read-only in prod, auto-expiring in 1h
scoped‑write incident — widen verbs, shorten TTL, pin the object
$ export KUBECONFIG="$(kubectl tessera \
    --verb get,list,watch,patch,delete \
    --resource pods --namespace prod \
    --resource-name stuck-pod-abc123 --ttl 20m --print-kubeconfig)"

Using --print-kubeconfig? It leaves the RBAC objects for the sweeper — make sure kubectl tessera gc (or the in‑cluster CronJob) runs.

How it stays safe

Least privilege, by construction

01

Pre‑flight gate

Every requested rule is checked with a SelfSubjectAccessReview. Ask for more than you hold and it aborts — printing an allowed/denied table — before creating anything.

02

Minted as you

RBAC objects are created with your credentials, never a privileged context or impersonation. That’s what makes the non‑admin boundary real.

03

Three‑layer cleanup

The token self‑expires, the interactive shell traps its exit, and tessera gc sweeps any orphans by label. Nothing lingers.
